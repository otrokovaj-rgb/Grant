# Grant
AI-система для автоматизации проверки грантовых заявок. Использует NLP и rule-based анализ для обработки финансовых отчётов и текстовых материалов. Сокращает время обработки, повышает объективность и прозрачность решений. 


## Краткое описание

Этот проект - **гибридная программная система на базе искусственного интеллекта**, предназначенная для автоматизации и оптимизации операционных этапов процесса рассмотрения и утверждения грантов. Система сочетает методы обработки естественного языка (NLP) и формальной логики для объективного анализа как финансовых отчётов, так и творческих материалов (сценариев), поддерживая принятие решений экспертами.

**Основная цель:** сократить время обработки заявок, автоматизировать первичную проверку документов, снизить ручной труд сотрудников, сохранив качество отбора и обеспечив прозрачность процесса.   

**Суть идеи:** автоматизировать две ключевые задачи:

1. Проверка финансовых отчётов
Бумажные ежеквартальные отчёты:
- Автоматически распознавать сканы и проверять, сходятся ли суммы расходов с лимитами
- Контролировать, чтобы перерасход по отдельной статье не влиял на общий итог + перерасход не превышал 10% от всей субсидии

2. Анализ сценариев для детского контента
- Проверка сценариев на наличие недопустимых сцен (насилие, взрослый контент)

---

## Проблема, которую решает проект

В процессе рассмотрения грантовых заявок существуют **рутинные и трудоёмкие этапы**:
- Предварительная фильтрация заявок
- Проверка формальной корректности документов
- Валидация финансовых отчётов
- Оценка соответствия заявки ценностным и тематическим критериям

Это приводит к **затратам времени, человеческим ошибкам и непрозрачности процесса**. Наш проект предлагает **автоматизированное решение**, которое ускоряет эти процессы, сохраняя контроль экспертов («человек в петле»).


## Ключевые возможности системы

1. **Автоматическое извлечение данных из документов** (OCR + NER):
   - Распознавание текста из сканов и PDF
   - Извлечение ключевых сущностей (даты, суммы, названия организаций)

2. **Проверка финансовых отчётов на соответствие правилам**:
   - Rule-based модели для выявления аномалий и несоответствий
   - Автоматическая сверка данных

3. **Семантический анализ текстовых материалов**:
   - Оценка тематики и соответствия ценностным критериям
   - Классификация жанра/тематики (для сценариев)

4. **Поддержка принятия решений**:
   - Система предлагает метки «manual review» / «auto ok»
   - Объяснимые результаты проверок


## Архитектура и технологический стек

### Основные компоненты
- **Data Pipeline**: OCR → Текстовая нормализация → NER → Rule-based анализ → Семантический анализ
- **Core Modules**:
  - Модуль парсинга структурированных данных (таблицы, формы)
  - Модуль rule-based проверок
  - NLP-модуль для анализа текстов

### Технологии
- **Язык**: Python
- **OCR**: Tesseract (возможно?)
- **NLP**: spaCy, pymystem3
- **Контейнеризация**: Docker
- **Развёртывание**: REST API, облачные вычисления


## Метрики оценки эффективности

1. **Время обработки заявки** (сокращение на 30–50%)
2. **Точность обнаружения ошибок** (Precision/Recall)
3. **Коэффициент ложных срабатываний** (False Positive Rate)


## Дорожная карта проекта (6 месяцев)

| Этап | Срок | Задачи |
|------|------|--------|
| **Подготовка** | Месяц 0–1 | Сбор шаблонов документов, анализ требований |
| **Разработка ядра** | Месяц 1–2 | OCR + NER, EDA по историческим данным |
| **Разработка модулей** | Месяц 2–4 | Rule-based логика, NLP-модели |
| **Тестирование** | Месяц 4–5 | Тесты на исторических данных, настройка workflows |
| **Пилот** | Месяц 5–6 | Пилотное внедрение |
